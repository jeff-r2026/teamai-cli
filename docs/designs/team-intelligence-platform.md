## Overview

Transform TeamAI from a simple skill-sharing CLI into a **Team Intelligence Platform** — every AI coding session contributes to shared knowledge (especially AI tool misuse patterns), every skill has usage metrics, and the system actively helps people discover valuable tools.

## Engineering Decisions (from /plan-eng-review)

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| 1 | Session 数据收集方式 | Collect-then-summarize | Stop hook 不直接调 LLM，避免递归和 per-session token 成本 |
| 2 | Skill 使用追踪 | PostToolUse hook + JSONL | 追踪 Claude Code Skill 工具调用，JSONL 避免并发写入竞争 |
| 3 | Session 存储格式 | 按月聚合 MD 文件 | 平衡文件数量和可读性，10人团队90天≈30文件 |
| 4 | Hook 注入方式 | 扩展现有 hooks.ts | 最小 diff，复用现有模式 |
| 5 | 团队推送方式 | SessionStart hook 自动上报（随 pull 捎带） | 零用户操作，per-user 目录避免冲突，best-effort + 5s 超时 |
| 6 | Health Score 公式 | usage(0-60) + freshness(0-40) | 小团队 contributor 维度区分度低，简化为两维 |
| 7 | Session 内容来源 | AI 对话和工具使用（非 git diff） | 核心目的是记录 AI 用错工具的情况 |
| 8 | Session 过滤 | 有价值才推送（包含工具错误/重试→有价值） | 避免团队仓库充斥无用信息 |
| 9 | Hook 追踪范围 | 只追踪 Skill 工具调用 | 追踪所有工具会产生大量噪音数据，核心需求是 skill 使用统计 |
| 10 | JSONL 生命周期 | 上报成功后截断已上报数据 | 文件永远很小，不丢数据 |
| 11 | 错误处理策略 | 所有 I/O 操作 try-catch + graceful degrade | 零静默失败原则 |
| 12 | Feature 3 推送方式 | 删除 `push --stats`，只用 auto-report | Decision 5 已决定自动上报，手动命令冗余 |
| 13 | Hook 命令实现 | `teamai track` TypeScript CLI 命令 | 类型安全 + 输入验证，比 bash one-liner 稳健 |
| 14 | Git 直推实现 | 复用 `utils/git.ts` pushRepoDirectly() | DRY，不新建 git 逻辑 |

## Accepted Features

### Core Features

#### 1. Session 工具使用记录
**What:** 在会话内主动收集 AI 对话过程和工具使用记录，重点关注 AI 用错工具的情况。通过 `teamai save-session` 命令或 Stop hook 触发。

**How it works:**
```
会话进行中 → AI 使用工具、对话
                    │
                    ▼
        用户/AI 调用 teamai save-session
                    │
                    ▼
        收集工具使用记录 + 对话摘要
                    │
                    ▼
        评估价值（工具误用?重试?有趣发现?）
                    │
            ┌───────┴───────┐
            ▼               ▼
        有价值            无价值
        标记可推送        仅本地保留
            │
            ▼
    ~/.teamai/sessions/<year-month>.md (追加)
```

**Value evaluation rules (v1):**
- 有价值：包含工具错误、重试、回退、AI 自我纠正
- 有价值：发现新模式、创造性解法
- 无价值：纯聊天、简单文件操作、常规编辑

**Storage:** `~/.teamai/sessions/<year-month>.md` 按月聚合。

#### 2. Skill Usage Tracker (Local)
**What:** PostToolUse hook 检测 Claude Code 的 Skill 工具调用，追加写入 `~/.teamai/usage.jsonl`。

**Data format (JSONL, one event per line):**
```jsonl
{"skill":"code-review","timestamp":"2026-03-19T10:30:00Z","tool":"claude"}
{"skill":"tdd","timestamp":"2026-03-19T10:35:00Z","tool":"claude"}
```

**Aggregation:** `teamai stats` 命令读取 JSONL 并聚合显示。

**CLI output:**
```
Skill Usage Statistics:
  code-review    47 uses   last: 2h ago
  tdd            23 uses   last: yesterday
  plan-eng-review 5 uses   last: now
```

#### 3. Team Usage Aggregation
**What:** `teamai push --stats` 聚合本地 usage.jsonl 为 `stats/<user>.yaml`，直接 commit 到团队仓库（不创建 MR）。

**Push flow:**
```
teamai push --stats
    │
    ▼
聚合 ~/.teamai/usage.jsonl → stats/<user>.yaml
    │
    ▼
git pull → git add → git commit → git push (直接到 master)
    │
    失败 → retry 3 次 (pull-rebase)
```

### Expansion Features

#### 4. Skill Health Score
**What:** 简化的两维评分 = f(usage_frequency, last_used_freshness)。

**Formula:**
- usage_score: normalized weekly usage count (0-60 points)
- freshness_score: days since last used, decaying (0-40 points)
- Total: 0-100, displayed as ★★★★★ (5 stars)
- contributor_count: 仅作为展示信息，不纳入评分

#### 5. 团队工作日志聚合
**What:** `teamai push --sessions` 推送有价值的 session 摘要到团队仓库 `sessions/<user>/<year-month>.md`。直接 commit，不创建 MR。

**Retention:** Auto-cleanup of sessions older than 90 days.

#### 6. Skill Recommendations
**What:** After `teamai pull`, suggests skills that are popular with the team but the user hasn't tried yet.

**Output example:**
```
💡 Recommended skills (popular with your team):
  • code-review — used by 85% of team (you haven't tried it)
  • tdd — used by 70% of team
```

#### 7. Weekly Team Digest
**What:** `teamai digest` generates a formatted weekly summary of team activity.

**Output includes:**
- New/updated skills this week
- Most-used skills
- Session summary highlights (tool misuse patterns)
- Team activity stats

## Security

- Usage data: skill name + timestamp only, no conversation content in JSONL
- Session summaries: AI 工具使用模式，不包含具体代码或敏感数据
- Skill name validation (prevent path traversal)
- Sessions retention policy (90 days)
- JSONL: corrupted lines skipped gracefully on read

## NOT in Scope

- Contributor Leaderboard — 竞争性排名可能影响团队文化
- Cross-team skill marketplace — 需要跨团队认证/授权机制
- Real-time dashboards (web UI) — CLI 工具不需要 web 前端
- Auto-generated skill README — 低优先级
- ML-based recommendations — 简单频率推荐已足够
- Cursor PostToolUse hook — 等 Cursor hook API 稳定后再支持（已记录 TODO）
